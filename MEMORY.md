## 设计决策记录

### 为什么不用 `--resume`？

接管时启动新会话（不用 `--resume`）是核心设计决策：
- **问题根源：** 会话卡住通常因为上下文过长触发 API 限制（429/524）
- **解决方案：** 新会话从零开始，避免继承庞大上下文
- **信息传递：** 通过接管指令告知 sessionId，新会话按需读取旧 transcript

### 为什么只读不杀？

工具设计哲学：**只读监控 + 显式接管**
- 误杀风险：5 分钟无响应可能是正常的长任务
- 用户控制：所有干预动作需用户明确按键确认
- 数据安全：绝不自动修改或删除任何会话文件
