🚑 Claude Rescue

5分钟快速上手 - 监控并救援卡住的 Claude Code 会话

1

检查环境

确保你已经安装了 Node.js 16 或更高版本：

node --version

如果看到类似 v18.0.0 的输出，说明环境 OK！

💡 提示：如果没有 Node.js，访问 nodejs.org 下载安装。
2

获取工具

克隆或下载本项目：

git clone https://github.com/你的用户名/claude-rescue.git
cd claude-rescue

或者直接下载 ZIP 文件并解压。

3

启动工具

在项目目录下运行：

node sessions.js

你会看到一个漂亮的实时监控界面：

┌─────────────────────────────────────────────────┐
│ 🚀 Claude 会话监控    共 3 个 · 3 运行 · 1 异常 │
├─────────────────────────────────────────────────┤
│     名字      ID      状态   活动    目录        │
│ ▶ ● 项目A    52348f  错误   2分钟前  ~/project  │
│   ● 项目B    d8480e  空闲   7分钟前  ~/code     │
└─────────────────────────────────────────────────┘
💡 建议：在独立终端窗口运行，不要在 Claude 会话内运行。
4

基本操作

使用键盘快捷键控制界面：

  • ↑ ↓ 或 j k - 上下移动
  • Enter - 查看会话详情
  • o - 接管卡住的会话（重要！）
  • g - 手动刷新一次
  • a - 开关自动刷新（默认每2秒）
  • / - 按名字/ID/目录过滤
  • s - 切换排序方式
  • q - 退出
5

救援卡住的会话

当你发现某个会话卡住了（显示为红色或黄色），按 o 键进入接管向导：

  1. 确认接管：按 y 继续
  2. 是否跳过权限？：通常按 n（默认否）
  3. 是否命名新会话？：按 n 跳过，或按 y 后输入名字

工具会自动打开新终端，在相同目录启动一个全新的 Claude 会话来处理卡住的任务！

📊

状态指示说明

  • ● 绿色 - 正常运行
  • ● 黄色 - 重试中或无响应（超过5分钟无更新）
  • ● 红色 - 遇到错误（如 API 524 超时）
  • ○ 灰色 - 进程已退出
💡 提示：黄色和红色状态说明会话可能卡住了，是时候用 o 键接管救援！

高级用法

脚本化输出

# 纯文本表格
node sessions.js --list

# JSON 格式（供脚本使用）
node sessions.js --json

# 提取所有活跃会话的 ID
node sessions.js --json | jq -r '.[] | select(.alive) | .sessionId'

全局安装

npm install -g .
claude-rescue  # 之后可以在任何地方运行

自定义会话目录

node sessions.js --dir ~/my-custom-path/sessions
❓

常见问题

Q: 为什么看不到我的会话？

A: 确保你的 Claude Code 正在运行。会话目录只包含当前运行的会话，退出后文件会被自动删除。

Q: 可以重命名会话吗？

A: 不能在这个工具里重命名。请在 Claude Code 内使用 /rename <新名字> 命令。

Q: 接管会话会杀掉原来的进程吗？

A: 不会！接管是启动一个全新的 Claude 会话，原会话继续运行。新会话会读取卡住会话的进度并继续工作。

Q: 我的终端不支持颜色怎么办？

A: 使用 --no-color 参数：node sessions.js --no-color
